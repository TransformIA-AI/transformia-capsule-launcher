# Day-1 User Journey

## Purpose

C02 turns the public repository from strategy-only into a day-1 visitor surface that is understandable, safe and locally verifiable.

## Journey

1. A visitor arrives from GitHub, X, Product Hunt or a founder link.
2. The visitor sees the one-liner: run governed AI work capsules locally or in BYOK mode, with a clear path to TransformIA Cloud.
3. The visitor understands local/BYOK/control as a future user-owned-key and local-control path.
4. The visitor sees the source-available/IP boundary and understands this repository is public source-available, not open source.
5. The visitor clones the public repository.
6. The visitor runs `npm run -s quality`.
7. The visitor reads `QUICKSTART.md` and the boundary docs.
8. The visitor sees no false installer, payment, private-runtime or production claims.
9. The visitor understands the future TransformIA Cloud handoff path.
10. The visitor knows the next Atlas step is C03 public Capsule template manifests.

## Success criteria

- Product promise is understandable in 30-60 seconds.
- Clone and validation path is clear and under 5 minutes.
- Source-available/IP posture is visible.
- Private core boundary is explicit.
- Local/BYOK/control promise is clear without asking for keys.
- Install promise is visible without claiming an installer exists.
- Cloud path is visible without implementing Cloud.

## Failure modes to prevent

- Visitor thinks the private runtime is bundled.
- Visitor thinks an executable launcher exists in C02.
- Visitor thinks an installer or binary is available in C02.
- Visitor thinks payment, Stripe, customer portal, subscription state, auth, DB, telemetry or Cloud sync exists in this repo.
- Visitor thinks the custom source-available license grants broad production, commercial, redistribution, SaaS or white-label rights.
- Visitor is asked for API keys before a BYOK scaffold exists.

## What not to overbuild

- No installer.
- No binary assets.
- No fake UI or fake GIF.
- No templates before C03.
- No provider setup before a later Atlas-scoped BYOK config PR.
- No private-runtime implementation.

## CTA posture

Primary CTA: clone the public repo and run validation.

Secondary CTA: read `QUICKSTART.md`, `docs/LOCAL_BYOK_QUICKSTART.md`, `docs/INSTALL_PROMISE_BOUNDARY.md` and `docs/CLOUD_HANDOFF_BOUNDARY.md`.

Future CTA: follow C03 for public Capsule template manifests.
