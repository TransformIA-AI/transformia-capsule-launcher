## C01 meaning

C01 defines the public repository strategy.

It should create the documentation and validation boundary that separates:

- public launcher distribution;
- private TransformIA core;
- public templates;
- future installer/quickstart work;
- future Cloud/paid handoff.

## C01 does not mean

- no executable launcher;
- no installer;
- no binaries;
- no private runtime code;
- no production connector execution;
- no customer data;
- no secrets;
- no payment implementation;
- no Stripe;
- no auth;
- no database;
- no cloud sync;
- no telemetry.

## Recommended C01 output

- README skeleton;
- launcher public repo strategy doc;
- private core boundary doc;
- public templates boundary doc;
- install promise boundary doc;
- templates placeholder;
- validation script;
- package scripts;
- release/status doc.
