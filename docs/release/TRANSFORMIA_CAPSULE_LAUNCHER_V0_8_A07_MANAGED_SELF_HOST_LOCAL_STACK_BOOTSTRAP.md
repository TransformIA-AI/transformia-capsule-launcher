# TransformIA Capsule Launcher v0.8-A07 release note

## Purpose

This release note records the public-safe managed and self-host local stack bootstrap planning capability for Atlas item `v0.8-A07`.

## Exact Atlas item

`v0.8-A07` - add managed and self-host local stack bootstrap.

## What A07 builds

- Managed Cloud handoff stack manifest.
- Self-host local stack plan manifest.
- Bootstrap manifest schema.
- Public-safe example configs.
- Zero-dependency stack plan script.
- A07 validator integrated into the launcher quality chain.

## What A07 does not build

A07 does not build a production installer, binary release, real Cloud deployer, private runtime provisioner, connector executor, provider integration, database provisioner, webhook server or service runner.

## Managed mode

Managed mode is a future TransformIA Cloud handoff plan. It validates readiness shape and keeps customer credentials, private runtime operation and activation outside this public repo.

## Self-host mode

Self-host mode is a local plan only. Operators may review external prerequisites, but this launcher does not run Docker, start services, collect credentials, clone runtime code or connect providers.

## Private core boundary

Private core is not included. The release does not include private runtime source, proprietary orchestration logic, internal prompts, customer material, raw evidence or production workflows.

## Credential handling boundary

No credentials are requested or stored. The examples only use public-safe plan metadata and safety flags.

## No live boundary

No live execution is performed. Human approval is required before any future live activation.

## No binary installer

No binary, installer, archive or compiled artifact is generated by A07.

## No Cloud deploy

No Cloud infrastructure is deployed. Managed mode remains a handoff plan.

## No Docker or service start

The launcher does not start Docker, n8n, Supabase, databases, webhook servers, provider jobs or model gateways.

## Mode selection guardrails

The planner defaults to self-host only when neither `--mode` nor config mode is provided. A valid config `bootstrapMode` selects the managed or self-host manifest when `--mode` is absent. If `--mode` and config disagree, the planner exits non-zero. Unknown `--mode` values also exit non-zero.

## Operator next steps

Run the validator and planner commands, review boundaries, keep credentials outside Git and request Dani approval before future live activation.

## Future A08 dependency

A08 should introduce deploy doctor, rollback doctor and credential checks before any approved deployment or activation path proceeds.
