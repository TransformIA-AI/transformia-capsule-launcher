## Post 1

I am opening the public surface for TransformIA Capsule Launcher.

It is a source-available repo for explaining the future local/BYOK path for governed AI work capsules, without exposing the private TransformIA runtime.

## Post 2

The goal is not hype.

The goal is a clean public boundary:

- what exists today
- what is planned
- what stays private
- what must not be claimed before it ships

Governed AI needs honest launch surfaces.

## Post 3

What exists today:

- README and quickstart validation
- public-safe template manifest catalog
- placeholder-only local/BYOK config scaffold
- private-core boundary docs
- validators that block false launch claims

## Post 4

What does not exist yet:

- no executable launcher
- no installer
- no local capsule execution from this repo
- no configured provider path
- no runnable templates
- no private runtime included
- no payment or Cloud sync

That clarity is intentional.

## Post 5

The template manifests are previews of future Capsule directions.

They describe business jobs, conceptual inputs and safety boundaries. They do not include customer data, prompts, connector workflows or runtime logic.

## Post 6

The local/BYOK path is planned and documented.

C04 added a placeholder config scaffold so people can see where future local settings will live, while real secrets stay out of the repo and provider execution remains absent.

## Post 7

The license posture matters.

This is public source-available software under a custom evaluation license. Public visibility does not grant production, SaaS, redistribution, white-label or commercial usage rights.

## Post 8

Why make this public now?

Because adoption needs more than a landing page. Builders and operators should be able to inspect the repo, see the boundaries and challenge the roadmap before execution claims appear.
