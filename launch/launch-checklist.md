# Launch checklist

## Pre-launch

- Verify validators: run `npm run -s quality`.
- Verify license/IP posture remains `SEE LICENSE IN LICENSE.md` and source-available.
- Verify no secrets, private keys, real `.env` files or customer material exist.
- Verify no false claims about execution, install, payment, Cloud, providers, templates or private runtime inclusion.
- Verify C04 is merged and the local/BYOK config scaffold is present.
- Verify GitHub description is claim-safe.
- Verify repo topics if desired, without implying implementation that does not exist.

## Launch

- Publish GitHub post or release note only after human review.
- Post X thread only after human review.
- Post LinkedIn founder post only after human review.
- Product Hunt draft review before any external submission.
- Collect feedback through issues and direct review.

## Post-launch

- Triage issues for claim safety, docs clarity and roadmap signal.
- Capture objections from builders, operators and SMEs.
- Update FAQ and launch copy only through reviewed commits.
- Prepare C06 web CTA/download surface with the same honesty boundaries.
- Do not merge external contributions without IP review.
